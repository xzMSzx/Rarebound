PROJECT PATCH — PHASE 3.7 OVERLAY STATE GUARD

This patch fixes a race condition between the card reveal animation and the Skip Pack summary overlay.

The current implementation may still allow asynchronous reveal operations to write to the DOM after Skip Pack has already transitioned the UI into summary mode.

This can cause the overlay to freeze or render only a single card.

The fix introduces a simple overlay state machine so asynchronous operations cannot corrupt the UI.

---

CRITICAL RULE

Do NOT modify:

engine logic  
RNG system  
pity system  
pack slot probabilities  
Monte Carlo simulation  

Only modify overlay interaction logic.

---

GOAL

Ensure that once Skip Pack is pressed, the UI transitions into SUMMARY mode and all reveal animations stop affecting the DOM.

No animation coroutine should ever modify the UI after the summary is shown.

---

CHANGE 1 — ADD OVERLAY STATE VARIABLE

Create a state variable for the overlay.

Example:

overlayState

Possible values:

"idle"  
"revealing"  
"summary"

Initial state should be:

"idle"

---

CHANGE 2 — SET STATE DURING REVEAL

When a pack begins revealing cards:

set:

overlayState = "revealing"

All reveal animations should check this state before modifying the DOM.

Example concept:

if (overlayState !== "revealing") return

This ensures animations cannot continue once the summary screen takes over.

---

CHANGE 3 — SET STATE ON SKIP

When Skip Pack is pressed:

Immediately set:

overlayState = "summary"

before any DOM updates.

This locks the system into summary mode.

---

CHANGE 4 — GUARD REVEAL COROUTINES

Inside revealCard and any animation functions:

Before updating the card DOM, check:

if (overlayState !== "revealing") return

If summary mode is active, the animation exits immediately.

This prevents late animation writes.

---

CHANGE 5 — SUMMARY RENDERING

When Skip Pack is pressed:

render the summary using the full pack results array.

The summary renderer must not depend on reveal progress.

Example:

showSkipSummary(packResults)

Do not reference reveal index or animation state.

---

CHANGE 6 — TAP TO CLOSE

When the summary screen is closed:

reset:

overlayState = "idle"

This allows the next pack to start normally.

---

EXPECTED RESULT

When Skip Pack is pressed during any stage of the reveal:

• all reveal animations stop safely  
• the summary screen renders correctly  
• no partial card state appears  
• no DOM corruption occurs  
• tapping closes the summary normally  

The pack opening overlay should remain stable regardless of when Skip is pressed.

END PATCH
