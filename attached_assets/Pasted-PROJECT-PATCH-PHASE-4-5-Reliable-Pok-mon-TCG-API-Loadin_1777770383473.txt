PROJECT PATCH — PHASE 4.5  
Reliable Pokémon TCG API Loading + Evolving Skies Fix

This patch fixes cases where a set (especially Evolving Skies / swsh7) fails to load or loads with empty card pools.

The goal is to ensure that:

• the API request is always correct  
• large sets (200+ cards) load reliably  
• empty responses are detected  
• rate limiting is retried automatically  
• the UI never enables pack opening unless cards actually exist  

This patch modifies only data loading logic.

---

CRITICAL RULES

Do NOT modify:

engine RNG  
pack generation logic  
probability tables  
rarity tiers  
pity system  
card reveal animations  

Only modify:

data/setLoader.js  
data/cardPoolManager.js  
main.js  

---

SECTION 1 — STANDARDIZE API REQUEST

File:

data/setLoader.js

Replace the fetch URL with the official Pokémon TCG API format.

const url = https://api.pokemontcg.io/v2/cards?q=set.id:${setId}&pageSize=250

Important:

pageSize must be 250 because some sets contain over 200 cards.

Example:

Evolving Skies → 237 cards

Without pageSize the API may return partial data.

---

SECTION 2 — ADD SAFE FETCH WRAPPER

Wrap the request with error handling.

Example:

async function fetchCards(url) {

try {

const response = await fetch(url)

if (!response.ok) {
throw new Error(API request failed: ${response.status})
}

const data = await response.json()

if (!data || !data.data || data.data.length === 0) {
throw new Error("API returned empty card list")
}

return data.data

}

catch (err) {

console.error("Card API error:", err)

throw err

}

}

---

SECTION 3 — ADD RETRY SYSTEM

Large sets occasionally fail due to API rate limiting.

Implement automatic retry.

Example:

async function fetchWithRetry(url, retries = 3) {

for (let i = 0; i < retries; i++) {

try {

return await fetchCards(url)

}

catch (err) {

console.warn(Retry attempt ${i + 1} failed)

if (i === retries - 1) throw err

await new Promise(r => setTimeout(r, 1000))

}

}

}

Use this instead of the original fetch.

---

SECTION 4 — VERIFY CARD IMAGES EXIST

Before returning card data, filter out cards with missing artwork.

Example:

const cards = data.data.filter(card => card.images && card.images.small)

If the result is empty:

throw new Error("No valid card images returned")

This prevents packs opening with blank cards.

---

SECTION 5 — UPDATE SET LOADER RETURN

The loader should now return:

return cards

instead of returning raw API response.

---

SECTION 6 — VALIDATE POOLS BEFORE ACTIVATION

File:

data/cardPoolManager.js

After loading cards and mapping rarities, verify that each pool contains cards.

Example:

if (
pool.common.length === 0 &&
pool.uncommon.length === 0 &&
pool.rare.length === 0
) {
throw new Error("Card pools empty after loading set")
}

This prevents the simulator from activating a broken set.

---

SECTION 7 — DISABLE PACK OPENING ON FAILURE

File:

main.js

Inside the set selection handler:

btnOpenPack.disabled = true

Only re-enable the button after the set loads successfully.

Example:

try {

await cardPoolManager.loadSet(setId)

btnOpenPack.disabled = false

}

catch (err) {

console.error("Set failed to load:", err)

showStatus("Failed to load set. Tap to retry.")

btnOpenPack.disabled = true

}
