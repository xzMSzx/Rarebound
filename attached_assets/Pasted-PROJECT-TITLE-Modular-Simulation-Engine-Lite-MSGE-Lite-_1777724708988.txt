PROJECT PURPOSE

This project builds a modular simulation engine that can run rule-based simulations.

The first simulation implemented will be a Trading Card Game (TCG) pack opening simulator inspired by Pokémon-style booster packs.

The engine must demonstrate:

modular architecture  
probabilistic systems  
simulation loops  
state-driven systems

The simulation engine must be reusable so that other simulations could be added later.

---

VERY IMPORTANT ARCHITECTURE RULE

The simulation engine must NOT depend on the UI.

The UI can read engine data.

But the engine must not contain any DOM manipulation.

Example:

Correct:

engine produces simulation state  
UI reads state and displays it

Incorrect:

engine directly updates HTML elements

ENGINE ARCHITECTURE

The engine must be modular and separated into layers.

---

LAYER 1 — ENGINE CORE

Create an Engine class.

Responsibilities:

store simulation state  
execute simulation steps  
run rules  
record event logs

Functions required:

initializeSimulation()

Creates initial simulation state.

stepSimulation()

Runs one simulation step.

runBatchSimulation(n)

Runs stepSimulation n times.

resetSimulation()

Resets state back to initial values.

The engine should maintain:

state  
entities  
rules  
eventLog

---

LAYER 2 — ENTITY SYSTEM

Entities represent objects in the simulation.

Examples:

Player  
Card  
Pack

Entity format example:

{
 id: "card_1042",
 type: "card",
 data: {
  rarity: "rare"
 }
}

Entities should be stored in an array.

---

LAYER 3 — RULES SYSTEM

Rules are functions that modify the simulation state.

Rules must follow this format:

function rule(state, engine)

Rules must return the updated state.

Example rules:

openPackRule  
generateCardRule  
applyPityRule  
updateStatisticsRule

Rules must not interact with the UI.

---

LAYER 4 — RNG SYSTEM

Create a dedicated RNG module.

File:

engine/rng.js

Functions required:

weightedRandom(options)

Example input:

[
{ value: "common", weight: 70 },
{ value: "rare", weight: 25 },
{ value: "epic", weight: 4 },
{ value: "legendary", weight: 1 }
]

The function returns one selected value.

---

SEEDED RANDOMNESS

Add seeded RNG capability.

The engine should contain a seed value.

Each random roll increments the seed.

This allows deterministic simulations.

Example:

engine.seed = 12345

This ensures the same simulation can be reproduced.

SIMULATION IMPLEMENTATION

Create one simulation:

Pack Opening Simulator

File:

---

PACK STRUCTURE

Each pack contains exactly 5 cards.

Slots:

Slot 1 — Common  
Slot 2 — Common  
Slot 3 — Common  
Slot 4 — Rare Slot  
Slot 5 — Hit Slot

---

RARITY PROBABILITIES

Rare Slot probabilities:

Rare = 80%  
Epic = 18%  
Legendary = 2%

Hit Slot probabilities:

Common = 90%  
Rare = 8%  
Epic = 1.8%  
Legendary = 0.2%

Common slots always produce Common cards.

---

PITY SYSTEM

Implement a pity mechanic.

Rule:

If the player opens 10 packs without receiving an Epic or Legendary card, the next pack guarantees an Epic.

Implementation steps:

Maintain pityCounter.

Increase pityCounter after each pack that contains no Epic or Legendary.

Reset pityCounter to zero when Epic or Legendary appears.

When pityCounter reaches 10:

Override Slot 4 rarity to Epic.

Log message:

"PITY SYSTEM ACTIVATED"

---

PACK OPENING PROCESS

When stepSimulation runs:

1. increment pack counter  
2. generate 5 cards based on slot rules  
3. create card entities  
4. update rarity statistics  
5. apply pity system  
6. update state values  
7. record log messages
