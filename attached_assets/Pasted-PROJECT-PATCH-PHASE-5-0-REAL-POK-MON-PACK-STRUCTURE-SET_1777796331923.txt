PROJECT PATCH — PHASE 5.0
REAL POKÉMON PACK STRUCTURE + SET REGISTRY

Goal:

Upgrade the pack simulator so booster packs follow the **real Pokémon TCG pack structure**.

Current packs randomly distribute rarities across slots.

This patch introduces:

• correct 10-card slot structure
• reverse holo slot
• dedicated hit slot
• central set registry (prevents ID mismatch bugs)

The engine RNG system must remain unchanged.

---

CRITICAL RULES

Do NOT modify:

engine RNG implementation
pity counter system
overlay animation logic
swipe controller
card reveal animator

Only modify:

data/setRegistry.js (new)
data/setProbabilityTables.js
data/cardPoolManager.js
simulations/packSimulation.js

SECTION 3 — IMPLEMENT REVERSE HOLO SLOT

Reverse holo can be drawn from:

common
uncommon
rare

Add property:

isReverseHolo: true

Example card structure:

{
name,
rarityType,
imageUrl,
isReverseHolo: true
}

If card is reverse holo, the renderer will later apply a shimmer effect.

SECTION 4 — IMPLEMENT HIT SLOT

Slot 10 uses probability tables.

Allowed rarities:

doubleRare
illustrationRare
ultraRare
specialIllustrationRare
hyperRare

Example logic:

const roll = Math.random()

if (roll < table.hyperRare) return "hyperRare"
if (roll < table.specialIllustrationRare) return "specialIllustrationRare"
if (roll < table.ultraRare) return "ultraRare"
if (roll < table.illustrationRare) return "illustrationRare"

return "doubleRare"

Use existing probability tables.
