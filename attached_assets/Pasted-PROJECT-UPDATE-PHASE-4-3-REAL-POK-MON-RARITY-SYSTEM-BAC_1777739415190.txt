PROJECT UPDATE — PHASE 4.3 REAL POKÉMON RARITY SYSTEM (BACKBONE)

This update replaces the old simulator rarity system (common / rare / epic / legendary) with the real Pokémon TCG rarity structure.

The goal is to create a proper backbone for realistic booster pack generation.

This update does NOT change UI symbols yet. It only changes the internal rarity structure and probability logic.

Symbols will be introduced later once the rarity system is stable.

---

CRITICAL RULE

Do NOT modify:

engine RNG system  
pity system logic  
overlay animation system  
skip summary logic  
swipe controller  
card reveal animation system  

Only modify the Pokémon card data layer and pack generation logic.

---

SECTION 1 — REMOVE OLD SIMULATOR TIERS

Remove the following rarity categories from the system:

common  
rare  
epic  
legendary  

These categories should no longer exist anywhere in the data layer.

Define the following internal rarity categories.

These should be the new rarity identifiers used by the system.

SECTION 3 — UPDATE RARITY MAPPER

File: data/rarityMapper.js

Modify mapPokemonRarity() so that Pokémon API rarity strings map to the new categories.

Mapping table:

Common → common

Uncommon → uncommon

Rare → rare

Rare Holo → holoRare

Rare Holo EX → doubleRare  
Rare Holo GX → doubleRare  
Rare Holo V → doubleRare  
Rare Holo VMAX → doubleRare  
Double Rare → doubleRare

Illustration Rare → illustrationRare

Ultra Rare → ultraRare

Special Illustration Rare → specialIllustrationRare

Hyper Rare → hyperRare

Secret Rare → hyperRare

If rarity is missing:

default to common

SECTION 6 — UPDATE PACK STRUCTURE (10 CARD PACK)

Update pack generation so each pack contains:

4 commons  
3 uncommons  
1 reverse slot  
1 rare slot  
1 hit slot

Structure:

slot1 → common  
slot2 → common  
slot3 → common  
slot4 → common  

slot5 → uncommon  
slot6 → uncommon  
slot7 → uncommon  

slot8 → reverse (rare or holo)

slot9 → rare or holoRare

slot10 → hit slot

SECTION 7 — HIT SLOT LOGIC

The hit slot should use the set-specific probability table.

Example:

roll = Math.random()

Use cumulative probability to determine rarity.

Example:

if roll < holoRare → holoRare  
else if roll < holoRare + doubleRare → doubleRare  
else if roll < ... → illustrationRare  
etc

Then draw a random card from the corresponding pool.
