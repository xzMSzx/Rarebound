PROJECT PATCH — PHASE 3.10 FRESH TAP REQUIREMENT

This patch fixes the final issue where the skip summary closes instantly due to pointer events that were already in progress when the summary overlay appears.

The previous debounce solution prevented bubbling events, but some mobile browsers may still deliver the pointerup event to the newly rendered overlay element.

The fix requires a full fresh tap (pointerdown followed by pointerup) after the summary appears.

---

CRITICAL RULE

Do NOT modify:

engine logic
RNG system
pity system
pack slot probabilities
Monte Carlo simulation

Only modify the overlay tap handling logic.

---

PROBLEM

When Skip Pack is pressed:

1. The summary overlay renders.
2. A pointerup event that started on the Skip button may be delivered to the overlay.
3. The overlay's tap handler interprets this as a valid tap.
4. The summary closes immediately.

Even with debounce delays, some browsers replay pointer events to the element currently under the pointer.

---

GOAL

The summary should only close after the user performs a new tap that begins after the summary appears.

A leftover pointerup event must never close the overlay.

---

CHANGE 1 — TRACK POINTERDOWN STATE

Inside waitForTap, introduce a flag:

let pointerDownSeen = false

---

CHANGE 2 — POINTERDOWN HANDLER

Attach a pointerdown listener to the overlay.

When pointerdown occurs:

pointerDownSeen = true

This indicates the user has started a fresh tap after the summary appeared.

---

CHANGE 3 — POINTERUP HANDLER

Modify the pointerup handler.

Before closing the overlay, check:

if (!pointerDownSeen) return

If no pointerdown has occurred since the summary appeared, ignore the event.

---

CHANGE 4 — CLOSE AFTER VALID TAP

If pointerDownSeen is true:

remove listeners
close overlay
reset state

---

CHANGE 5 — RESET STATE

When the overlay returns to idle:

pointerDownSeen = false

This ensures the next pack starts clean.

---

EXPECTED RESULT

When Skip Pack is pressed:

• the summary appears
• leftover pointer events do nothing
• the overlay remains visible
• the user must perform a new tap (down + up) to close the summary

The summary will no longer disappear instantly.
