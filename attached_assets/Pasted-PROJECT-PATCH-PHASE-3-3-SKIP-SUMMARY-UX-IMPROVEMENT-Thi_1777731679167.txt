PROJECT PATCH — PHASE 3.3 SKIP SUMMARY UX IMPROVEMENT

This update improves the Skip Pack summary experience during pack opening.

The simulation engine must remain unchanged.

Only UI behavior and layout should be modified.

---

CRITICAL RULE

Do NOT modify:

engine logic
RNG system
pity system
pack slot probabilities
Monte Carlo simulation logic

Only modify UI components related to the pack reveal overlay.

---

GOAL

Improve the Skip Pack summary screen so that:

• cards are larger and easier to see
• the summary remains visible until the user taps
• the interaction feels intentional and readable

---

CHANGE 1 — CREATE DEDICATED SUMMARY LAYOUT

Currently the summary reuses card grid styling, causing cards to appear too small.

Create a dedicated container for skip summaries.

Example container name:

skip-summary-container

This container should display all cards horizontally.

---

SUMMARY CARD SIZE

Cards inside the skip summary should be significantly larger than grid cards.

Recommended dimensions:

width: 70px–90px
height: proportional to trading card ratio

Cards should be evenly spaced across the screen.

Example visual layout:

[C]   [C]   [C]   [R]   [E]

---

CARD STYLING

Each card should still display its rarity styling.

Common → grey border
Rare → blue glow
Epic → purple glow
Legendary → gold glow

Reuse existing rarity classes if available.

---

CHANGE 2 — REMOVE AUTO-CLOSE TIMER

Currently the summary closes automatically after approximately one second.

Remove the automatic closing behavior.

The summary must remain visible until the user interacts.

---

NEW CLOSE BEHAVIOR

After Skip Pack is pressed:

1. reveal all remaining cards
2. display the skip summary container
3. wait for user input

The summary should close only when the user taps the screen.

Pseudo behavior:

tap anywhere → close summary → return to main UI

---

CHANGE 3 — ADD INSTRUCTION TEXT

Below the summary cards display a message:

Tap anywhere to continue

---

INSTRUCTION STYLE

Position:

centered below cards

Color:

#cfcfcf

Font size slightly smaller than card counter.

Add subtle opacity to avoid visual clutter.

Example concept:

opacity: 0.8

---

CHANGE 4 — CENTER SUMMARY CONTENT

Ensure the summary cards appear centered vertically and horizontally within the overlay.

Example layout structure:

overlay container
summary container
card row
instruction text

Spacing between cards should be consistent.

Example:

gap: 14px

---

CHANGE 5 — PREVENT SWIPE INTERACTION DURING SUMMARY

When the skip summary is visible, swipe detection should be disabled.

Only tap input should close the summary.

---

EXPECTED RESULT

After pressing Skip Pack:

• all pack cards appear in a larger row
• the cards are clearly visible
• the screen remains until the user taps
• tapping closes the summary and returns to the main interface

The pack opening experience should feel smoother and more readable.

END OF PHASE
