PROJECT UPDATE — RNG SEED IMPROVEMENT

This is a small update to the existing Modular Simulation Engine Lite project.

IMPORTANT:
Do NOT rebuild the project.
Do NOT change the architecture.
Do NOT modify unrelated files.

Only update the RNG seeding behavior.

The goal is to support BOTH:

1) random seeds for normal usage  
2) manual seeds for deterministic simulations

The engine should default to random seeds but allow developers to supply a specific seed.

---

CURRENT BEHAVIOR (PROBLEM)

The engine currently initializes with a fixed seed:

engine.seed = 12345

This causes every simulation reset to produce identical results.

Example:

Run 100 packs → results A  
Reset  
Run 100 packs → results A again

This is useful for debugging but not for normal simulation.

---

DESIRED BEHAVIOR

Engine should use a random seed by default.

Example:

engine.seed = Date.now()

This ensures each simulation run produces a different sequence of random numbers.

However, developers should still be able to pass a specific seed manually.

Example:

new Engine(12345)

This produces deterministic simulations.

---

IMPLEMENTATION REQUIREMENTS

Modify the Engine class constructor.

Current constructor likely looks similar to:

constructor() {
 this.seed = 12345
}

Replace it with:

constructor(seed = Date.now()) {
 this.seed = seed
}

Explanation:

If no seed is provided → use timestamp  
If seed is provided → use that seed

---

SEED RESET BEHAVIOR

When resetSimulation() is called:

The seed should NOT be reset to a fixed number.

Instead:

Option A (recommended)

Generate a new random seed.

Example:

this.seed = Date.now()

This ensures each reset produces new randomness.

---

LOGGING REQUIREMENT

When the engine initializes, log the seed being used.

Example log message:

"Simulation seed: 1714839021342"

This helps debugging and reproducibility.

---

EXAMPLE WORKFLOW AFTER UPDATE

Case 1 — normal usage

User opens the app.

Engine initializes automatically:

seed = Date.now()

Each reset produces different simulation results.

---

Case 2 — deterministic debugging

Developer manually initializes engine with seed.

Example:

const engine = new Engine(12345)

Now simulation becomes reproducible.
