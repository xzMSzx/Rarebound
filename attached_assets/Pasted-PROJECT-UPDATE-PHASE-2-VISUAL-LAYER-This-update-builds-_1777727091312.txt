PROJECT UPDATE — PHASE 2 VISUAL LAYER

This update builds the user-facing visual experience for the Modular Simulation Engine Lite project.

Phase 1 has already implemented:

simulation engine  
pack generation rules  
seeded RNG  
pity system  
batch simulation  
Monte Carlo simulation  
slot analytics

These systems are working correctly and must not be modified.

---

CRITICAL RULE

Do NOT modify the simulation engine.

Do NOT change:

engine logic  
probability rules  
pack slot design  
pity system  
Monte Carlo simulation

The engine must remain exactly as it is.

All Phase 2 work must occur in UI modules only.

---

PROJECT GOAL

Add visual systems that transform the simulation into an interactive pack opening experience.

The application should now support:

animated pack opening  
visual card rendering  
rarity glow effects  
live probability graphs  
dramatic reveal suspense

The simulation logic remains unchanged.

---

NEW UI FEATURES

Implement the following systems:

1) Card Reveal Animation System  
2) Card Grid Renderer  
3) Probability Convergence Graph  
4) Pack Reveal Flow Controller  
5) Suspense Detection (TCG-style feature)

---

FILE STRUCTURE

Add new files:

ui/

animationSystem.js  
cardRenderer.js  
probabilityGraph.js  
packRevealController.js

Existing UI renderer should remain but may call these modules.

---

FEATURE 1 — CARD REVEAL ANIMATION

Purpose:

Cards should reveal sequentially with a flip animation.

Steps for each card:

1) card back appears  
2) short delay  
3) card flips  
4) rarity glow appears

Delay between cards:

300 milliseconds

---

CARD BACK DESIGN

Before reveal, each card should show a generic card back.

Example:

dark rectangle  
question mark or card icon

---

FLIP ANIMATION

Use CSS transforms.

Example concept:

transform: rotateY(180deg)

Duration:

300ms

---

RARITY GLOW EFFECTS

When a card is revealed, apply glow depending on rarity.

Common:

subtle grey border

Rare:

blue glow

Epic:

purple glow

Legendary:

gold glowing border with pulsing animation

Legendary should appear visually special.

---

FEATURE 2 — CARD COLLECTION GRID

Purpose:

Display all cards the player has obtained.

Cards should appear in a grid.

Example layout:

5 cards per row

Each card displays:

rarity color  
pack number

The grid updates whenever new cards are generated.

---

CARD COLOR SCHEME

Common:

grey background

Rare:

blue background

Epic:

purple background

Legendary:

gold background

---

FEATURE 3 — PROBABILITY CONVERGENCE GRAPH

Purpose:

Visualize how real drop rates approach expected probabilities.

Use Chart.js.

Graph type:

line chart

---

GRAPH AXES

X axis:

packs opened

Y axis:

drop rate percentage

---

GRAPH DATA

Display lines for:

Common  
Rare  
Epic  
Legendary

Graph updates whenever packs are opened.

---

GRAPH UPDATE RULE

After every pack opening:

recalculate actual rarity percentages

update graph dataset

render updated chart

---

FEATURE 4 — PACK REVEAL FLOW CONTROLLER

Purpose:

Control the sequence of card animations during pack opening.

Location:

ui/packRevealController.js

---

REVEAL SEQUENCE

When user clicks Open Pack:

1) retrieve card results from engine
2) create temporary card back elements
3) reveal cards sequentially
4) update card grid after reveal
5) update probability graph

---

SEQUENCE TIMING

Card reveal delay:

300 milliseconds

Total pack animation time:

~1.5 seconds

---

FEATURE 5 — SUSPENSE DETECTION (TCG FEATURE)

Purpose:

Add dramatic tension when a high rarity card is present.

Detect if the pack contains:

Epic or Legendary

---

SUSPENSE EFFECTS

If a pack contains Epic or Legendary:

apply suspense behavior.

---

SUSPENSE VISUALS

Before the reveal:

card back glows faintly

during reveal:

card slightly shakes

flip animation slightly slower

---

LEGENDARY SPECIAL EFFECT

Legendary cards should:

emit golden particles or glow  
pulse for 2 seconds

This makes legendary pulls feel exciting.
