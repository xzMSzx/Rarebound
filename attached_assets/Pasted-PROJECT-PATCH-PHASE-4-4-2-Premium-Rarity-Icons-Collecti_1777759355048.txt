PROJECT PATCH — PHASE 4.4.2  
Premium Rarity Icons + Collection Grid Fix + API Reliability

This patch improves the visual quality of rarity symbols so they resemble Pokémon TCG Pocket style icons, fixes symbol overflow inside the collection grid, and stabilizes the Pokémon TCG API loading for larger sets such as Evolving Skies.

This patch does NOT change any simulation logic.

---

CRITICAL RULES

Do NOT modify:

engine RNG  
pack generation logic  
rarity probability tables  
pity system  
simulation engine  
overlay animation system  

Only modify:

ui rendering  
collection grid layout  
rarity icon assets  
set loading reliability

---

SECTION 2 — UPDATE CARD RENDERER

File:

ui/cardRenderer.js

Import the icon system:

import { RARITY_ICONS } from "./rarityIcons.js"

Replace any symbol or letter badge logic with:

rarityContainer.innerHTML = RARITY_ICONS[card.rarityType]

This ensures all cards use the new SVG icons.

---

SECTION 3 — FIX COLLECTION GRID SYMBOL OVERFLOW

Currently 3-icon rarities overflow the card tile.

Modify style.css.

Locate the rarity container inside collection cards.

Add:

.rarity-stack {

display: grid  
grid-template-columns: repeat(2, auto)  
gap: 3px  
justify-content: center  
align-items: center  

}

This allows:

1 icon  
2 icons  
3 icons  
future 5 icons  

to scale correctly without clipping.

SECTION 6 — FIX EVOLVING SKIES API LOADING

File:

data/setLoader.js

Replace fetch URL with:

https://api.pokemontcg.io/v2/cards?q=set.id:${setId}&pageSize=250

Add response validation:

if (!data || !data.data || data.data.length === 0) {

console.error("Set failed to load:", setId)

throw new Error("Card data failed to load")

}

This prevents empty card pools from loading silently.

SECTION 7 — ADD LOADING GUARD

In main.js after set loading:

if (!cardPoolManager.isSetLoaded()) {

disableOpenPackButton()

showError("Set failed to load")

}

This prevents opening packs with empty sets.
