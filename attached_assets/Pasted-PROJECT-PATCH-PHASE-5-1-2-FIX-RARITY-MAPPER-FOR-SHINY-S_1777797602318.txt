PROJECT PATCH — PHASE 5.1.2
FIX RARITY MAPPER FOR SHINY SETS

Goal:

Certain Pokémon sets (e.g. Paldean Fates) contain rarity strings such as:

"Shiny Rare"
"Shiny Ultra Rare"
"Shiny Rare V"
"Shiny Rare VMAX"
"Radiant Rare"

These are not currently recognised by data/rarityMapper.js and fall through to the default fallback:

?? 'common'

This causes rare cards to be incorrectly placed in the commons pool, corrupting pack slot behaviour.

This patch adds the missing rarity mappings.

---

CRITICAL RULES

Do NOT modify:

pack slot structure
RNG system
pity logic
card pool generator
renderer code

Only modify:

data/rarityMapper.js

---

SECTION 1 — LOCATE RARITY_MAP

Open:

data/rarityMapper.js

Find the object similar to:

const RARITY_MAP = {

'Common': 'common',
'Uncommon': 'uncommon',
'Rare': 'rare',
'Rare Holo': 'holoRare',
'Rare Holo V': 'doubleRare',
'Rare Holo VMAX': 'doubleRare',
'Rare Holo VSTAR': 'doubleRare',

...

}

---

SECTION 2 — ADD SHINY SET RARITIES

Add the following mappings:

'Shiny Rare': 'illustrationRare',
'Shiny Ultra Rare': 'specialIllustrationRare',
'Shiny Rare V': 'specialIllustrationRare',
'Shiny Rare VMAX': 'specialIllustrationRare',

'Radiant Rare': 'illustrationRare',

---

SECTION 3 — ENSURE FALLBACK REMAINS

Do NOT remove the fallback logic.

Example expected code:

return RARITY_MAP[apiRarity] ?? 'common'

This ensures the engine never crashes if a future set introduces a new rarity string.

---

SECTION 4 — ADD DIAGNOSTIC LOGGING

Add a warning when the fallback is triggered.

Example:

if (!RARITY_MAP[apiRarity]) {
console.warn("Unknown rarity string:", apiRarity)
}

This helps detect unsupported rarities when new sets are added.

---

SECTION 5 — VERIFY RESULT

Reload the simulator and open packs from Paldean Fates.

Expected pool distribution:

Commons ~60–70
Uncommons ~40–50
Rares ~20–30
Higher tiers correctly distributed

EX or full-art cards must **no longer appear in common slots**.

---

SECTION 6 — FUTURE PROTECTION

Because different Pokémon sets introduce unique rarity names, the mapper should always be updated when adding a new set.

Unknown rarity logs will make this easy to detect.

END PATCH
