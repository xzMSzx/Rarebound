PHASE 8 — COLLECTION IDENTITY, BINDER EVOLUTION, CARD DETAIL EXPERIENCE, WISHLISTS, STATS, AND LONG-TERM RETENTION

IMPORTANT:
This is NOT a visual-effects phase.
This phase transforms the app from:
“pack opening simulator”
into:
“real collectible collection experience.”

DO NOT rewrite or break:
- pack opening flow
- rarity FX
- holo systems
- sound systems
- preload systems
- reveal systems
- binder persistence
- collection persistence
- screen navigation

All Phase 8 systems must EXTEND the current architecture cleanly.

========================================================
SECTION 0 — PHASE GOALS
========================================================

This phase should psychologically create:

1. Ownership
2. Completion goals
3. Collection identity
4. Chase-card behavior
5. Retention loops
6. “Just one more pack” feeling

The UI should feel:
- premium
- collectible-focused
- emotionally rewarding
- not cluttered
- psychologically satisfying

Avoid:
- arcade feel
- game-y coin systems
- noisy dashboards
- overcrowded overlays

========================================================
SECTION 1 — COMPLETE BINDER OVERHAUL
========================================================

Current binder is functional but still feels like:
“a grid of images.”

We are converting it into:
“A real collectible binder.”

========================================================
1.1 — BINDER PAGE DESIGN
========================================================

KEEP:
- 3x3 layout
- pagination
- unknown slots

BUT improve the visual psychology.

Each page should resemble:
- matte-black premium binder paper
- soft recessed card pockets
- subtle inner shadows
- realistic slot spacing
- elegant page margins

Styling goals:
- feel physical
- feel premium
- no bright outlines
- no cheap neon UI

========================================================
1.2 — EMPTY SLOT DESIGN
========================================================

Unknown slots should NOT feel dead.

Current:
- plain dark rectangle
- question mark

Improve to:
- faint textured pocket
- soft dashed edge
- centered subtle Pokéball silhouette
- very faint card-number placeholder

Example:
#001
?
#002
?
etc.

This creates:
“missing collectible anticipation.”

========================================================
1.3 — SET COMPLETION HEADER
========================================================

Current top bar:
“Lost Origin 19 / 217”

Expand into:
- Set logo/title
- collected count
- completion percentage
- rarity progress

Example:

Lost Origin
19 / 217 • 8%

Secret Rares: 0 / 22

Very subtle typography.

========================================================
1.4 — PAGE FLIP FEEL
========================================================

When pressing next/previous page:

DO NOT instantly swap grids.

Instead:
- page slides horizontally
- slight fade
- 180–220ms duration
- subtle easing

Should feel like:
turning a real binder page.

========================================================
1.5 — CARD OWNERSHIP GLOW
========================================================

Owned cards should feel satisfying inside the binder.

Apply:
- soft rarity-dependent glow
- VERY subtle
- not flashy

Examples:
common:
- nearly none

rare:
- faint silver rim

ultra rare:
- faint chromatic edge glow

Only visible when focusing on the binder.

========================================================
SECTION 2 — COMPLETE CARD DETAIL EXPERIENCE
========================================================

Current modal is:
“large image + title.”

We are converting this into:
“A premium collectible inspection screen.”

========================================================
2.1 — CARD DETAIL LAYOUT
========================================================

Modal structure:

TOP:
- enlarged card
- tilt active
- holo active
- rarity glow active

BOTTOM PANEL:
Scrollable information area.

This prevents clutter.

========================================================
2.2 — DETAIL PANEL CONTENT
========================================================

Information order:

1. Card name
2. Rarity classification
3. Set + card number
4. Estimated market value
5. Pull rate
6. Evolution stage
7. Type
8. HP
9. Flavor text
10. Artist
11. Wishlist toggle

Use elegant spacing.

DO NOT cram everything tightly.

========================================================
2.3 — DYNAMIC MARKET VALUE SYSTEM
========================================================

VERY IMPORTANT:
We are PREPARING for Phase 9 economy.

We do NOT need real live card-market APIs yet.

Instead:
Create a procedural market-value engine.

========================================================
MARKET VALUE FORMULA
========================================================

Generate estimated value from:

- rarity tier
- pull rate
- set age multiplier
- special card type
- randomness band

Examples:

common:
$0.05–$0.40

rare:
$0.50–$4

double rare:
$5–$20

illustration rare:
$15–$60

special illustration:
$80–$300

hyper rare:
$120–$500+

This is NOT static.

Values should vary slightly per card.

========================================================
VALUE GENERATION REQUIREMENTS
========================================================

Store generated value persistently.

Meaning:
If a card generates:
$128.42

it ALWAYS remains:
$128.42

until future economy systems alter it.

Do NOT regenerate every render.

Persist:
- by card ID
- in local storage/player state

========================================================
2.4 — PULL RATE DISPLAY
========================================================

Show:
“Estimated Pull Rate”

Examples:
1 in 12 packs
1 in 48 packs
1 in 250 packs

Derived from rarity.

This psychologically increases:
- perceived rarity
- emotional reward

========================================================
2.5 — FLAVOR TEXT
========================================================

Extract flavor text dynamically from Pokémon TCG API card data where available.

If unavailable:
fallback gracefully.

Flavor text should:
- appear in smaller italic typography
- separated visually
- not dominate UI

========================================================
2.6 — TILT + HOLO PRESERVATION
========================================================

CRITICAL:
Card detail modal MUST preserve:
- holo shaders
- tilt interaction
- rarity glow
- sparkle FX

Do NOT downgrade the card into a static image.

This is the “showcase moment.”

========================================================
2.7 — MODAL UX
========================================================

Background:
- strong blur
- darkened overlay

Card:
- centered
- elevated
- shadowed

Open animation:
- scale 0.92 → 1
- opacity fade
- 180ms

Close:
- reverse

========================================================
SECTION 3 — COLLECTION PROGRESSION
========================================================

This is one of the most important retention systems.

========================================================
3.1 — SET COMPLETION TRACKING
========================================================

Each set tracks:
- owned cards
- missing cards
- rarity completion
- secret rares found

Persist permanently.

========================================================
3.2 — COMPLETION VISUALS
========================================================

Examples:

25%
- faint silver progress bar

50%
- subtle glow

75%
- animated shimmer

100%
- gold completion crown
- rainbow edge
- “SET COMPLETE”

This creates long-term goals.

========================================================
3.3 — DUPLICATE TRACKING
========================================================

Each card slot should show:
x2
x3
x5

very subtly.

Users psychologically enjoy:
“stacking duplicates.”

========================================================
SECTION 4 — WISHLIST / CHASE CARD SYSTEM
========================================================

Massive retention feature.

========================================================
4.1 — WISHLIST BUTTON
========================================================

Inside card detail modal:
“☆ Add to Wishlist”

When enabled:
- card becomes “chase card”

Persist permanently.

========================================================
4.2 — CHASE CARD VISUALS
========================================================

Wishlist cards:
- subtle gold bookmark corner
- star icon
- visible in binder

========================================================
4.3 — PACK REVEAL INTERACTION
========================================================

If a revealed card is wishlisted:

Before flip:
- subtle gold pulse
- slightly different audio cue
- tiny anticipation delay

DO NOT overdo this.

It should feel:
“Oh wait… this might be something.”

========================================================
SECTION 6 — UI/UX PREMIUMIZATION
========================================================

The app must now feel:
- cohesive
- premium
- collectible-focused

========================================================
6.1 — REMOVE HARSH EDGES
========================================================

Reduce:
- sharp outlines
- harsh borders
- bright blue strokes

Favor:
- subtle contrast
- layered depth
- shadows
- gradients

========================================================
6.2 — MOTION CONSISTENCY
========================================================

All transitions:
180–240ms.

Consistent easing throughout app.

No random timings.

========================================================
6.3 — SCROLL BEHAVIOR
========================================================

Pages that do not require scrolling:
- lock vertical scroll completely

No fake scrollbar movement.

========================================================
6.4 — HAPTIC FEEL (MOBILE)
========================================================

If supported:
- subtle vibration on ultra rare pulls
- tiny tap feedback on buttons

Must remain subtle.

========================================================
SECTION 7 — PERFORMANCE REQUIREMENTS
========================================================

DO NOT introduce:
- laggy binder renders
- rerender loops
- blocking API fetches

========================================================
7.1 — LAZY IMAGE LOADING
========================================================

Binder cards:
lazy load only visible page.

Do NOT render entire collection at once.

========================================================
7.2 — CACHE CARD DATA
========================================================

Card detail API data:
cache after first load.

Avoid repeated network fetches.

========================================================
SECTION 8 — FINAL PSYCHOLOGICAL GOAL
========================================================

After Phase 8:

Users should feel:

“I want to complete this set.”
“I want that chase card.”
“My collection actually matters.”
“My rare pulls feel important.”

NOT:
“I opened some random packs.”

That emotional shift is the ENTIRE purpose of this phase.
