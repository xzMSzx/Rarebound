PROJECT PATCH — PHASE 4.4.3 RARITY SYMBOL SYSTEM CORRECTION

This patch fixes the rarity symbol system so that:

• symbols render correctly in the overlay, collection grid, and rarity guide  
• premium rarities appear visually distinct  
• UR / SIR / HR symbols appear in the rarity guide  
• symbol styling actually applies

The previous patch only modified CSS but the UI still renders raw Unicode symbols.

This patch rewires the rendering pipeline.

---

CRITICAL RULES

Do NOT modify:

engine RNG  
pack generation logic  
pity system  
set probability tables  
simulation engine logic  

Only modify UI symbol rendering.

---

SECTION 1 — STANDARDIZE RARITY SYMBOL SOURCE

All UI must pull symbols from a single mapping.

File: ui/raritySymbols.js

Replace file contents with:

export const RARITY_SYMBOLS = {

common: "◇",

uncommon: "◇◇",

rare: "★",

holoRare: "★",

doubleRare: "★★",

illustrationRare: "★★",

ultraRare: "★★★",

specialIllustrationRare: "✦★",

hyperRare: "✦✦"

}

---

SECTION 2 — FORCE SYMBOL RENDERING IN CARD REVEAL

File: ui/fullscreenOverlay.js

Locate where rarity label is rendered.

Replace the entire block with:

const symbol = RARITY_SYMBOLS[card.rarityType] || "◇"

rarityLabel.innerHTML =
<span class="rarity-symbol">${symbol}</span>

rarityLabel.className = "rarity-label rarity-" + card.rarityType

This ensures styling classes apply.

---

SECTION 3 — FIX COLLECTION GRID SYMBOLS

File: ui/cardRenderer.js

Locate rarity badge logic.

Replace:

rarityBadge.textContent

with:

rarityBadge.innerHTML =
<span class="rarity-symbol">${RARITY_SYMBOLS[card.rarityType]}</span>

Also add:

gridCard.classList.add("rarity-" + card.rarityType)
